enable ~

  ‚Ä¢ type: `boolean`
    default: `true`

Self-explanatory.

wrap ~

  ‚Ä¢ type: `boolean`
    default: `true`

Enables `wrap` support, which causes block quote‚Äôs border to appear where the
text gets wrapped.

‚ñå üõë Caution
‚ñå Detecting where to show the border is done manually, sometimes it may be
‚ñå incorrect(e.g.¬†after resizing the Terminal).

default ~

  ‚Ä¢ type: `markview.config.markdown.block_quotes.opts`[^5]
    default[^6]

Configuration for regular block quotes.

  ‚Ä¢ type: `string`

Highlight group for `preview`.

  ‚Ä¢ type: `boolean`

enable ~

  ‚Ä¢ type: `boolean`
    default: `true`

Self-explanatory.

strict ~

  ‚Ä¢ type: `boolean`
    default: `false`

When enabled, leading & trailing whitespaces won‚Äôt be considered part of table
cell.

block_decorator ~

  ‚Ä¢ type: `boolean`
    default: `true`

Allows rendering top & bottom border of tables.

use_virt_lines ~

  ‚Ä¢ type: `boolean`
    default: `false`

Allows rendering `block_decorator` as virtual lines.

parts ~

  ‚Ä¢ type: `markview.config.markdown.tables.parts`[^124]
    default[^125]

Parts used to create the preview.

TOP

  ‚Ä¢ type: `string[]`

List containing 4 parts defining the top border. These parts are used for,

  ‚Ä¢ Left corner
  ‚Ä¢ Top
  ‚Ä¢ Right corner
  ‚Ä¢ Column separator

HEADER

  ‚Ä¢ type: `string[]`

List containing 3 parts defining the header‚Äôs border. These parts are used
for,

  ‚Ä¢ Left border
  ‚Ä¢ Column separator
  ‚Ä¢ Right border

SEPARATOR

  ‚Ä¢ type: `string[]`

List containing 4 parts defining the header & row separator. These parts are
used for,

  ‚Ä¢ Leftmost border
  ‚Ä¢ Horizontal border
  ‚Ä¢ Rightmost corner
  ‚Ä¢ Column separator

ROW

  ‚Ä¢ type: `string[]`

List containing 3 parts defining the row‚Äôs border. It has the same structure
as `header`.

BOTTOM

  ‚Ä¢ type: `string[]`

List containing 4 parts defining the bottom border. It has the same structure
as `top`.

hl ~

  ‚Ä¢ type: `markview.config.markdown.tables.parts`[^128]
    default[^129]

Highlight group for various parts. It has the same structure as `parts`.
